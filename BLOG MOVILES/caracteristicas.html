Estructura de Proyectos en Android Studio

Organización modular

En Android Studio, los proyectos se organizan mediante módulos, siendo el más común el módulo app. Esta estructura permite separar la lógica, los recursos y la configuración, facilitando la administración del proyecto.

Manifests

El archivo AndroidManifest.xml contiene la información esencial de tu aplicación para el sistema operativo: nombre de la app, permisos (como acceso a internet o cámara), actividades (pantallas) y servicios.

Java / Kotlin

En las carpetas java/ o kotlin/ reside el código fuente de la aplicación. Se divide en:

  • Nombre del paquete: lógica principal de la app (Activities, ViewModels, etc.).
  • androidTest: pruebas que corren en un dispositivo físico o emulador.
  • test: pruebas unitarias locales que corren en tu computadora.

Resources (res/)

Todo lo que no es código se guarda en la carpeta res/. Aquí se incluyen:

  • drawable: imágenes, íconos y vectores.
  • layout: archivos XML que definen la interfaz visual (si no usas Jetpack Compose).
  • mipmap: íconos de la aplicación para diferentes densidades de pantalla.
  • values: textos (strings.xml), colores (colors.xml) y estilos/temas.

Gradle Scripts

Gradle es el motor que compila y construye tu aplicación. Los archivos más importantes son:

  • build.gradle (Project): configuraciones globales de todo el proyecto.
  • build.gradle (Module: app): dependencias externas, versión mínima de Android y versión de la app.
  • settings.gradle: define qué módulos se incluyen en la compilación.
